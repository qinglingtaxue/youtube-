### 7. 真实数据调研（基于实际MCP调用）⭐
```bash
python3 examples/real_research.py
```
**功能**：
- 实际调用MCP工具收集数据
- 基于真实观看量、频道数、发布时间分析竞争度
- 动态评估不同品类的竞争情况
- 生成基于实际数据的分析报告

**使用步骤**：
1. 在Claude Code中执行MCP命令收集数据
2. 系统自动分析竞争度
3. 查看分析报告

**MCP调用示例**：
```bash
# 调研Python教程在新加坡的竞争情况
@playwright 打开 https://www.youtube.com/results?search_query=Python教程&gl=SG
@playwright 等待页面加载完成
@playwright 滚动页面到底部
@playwright 提取所有视频的标题、频道名、观看量、发布时间
@playwright 截图保存
```

**竞争度判断标准**：
- **观看量**: 平均观看量≥100万=高竞争，10-100万=中等，<10万=低竞争
- **频道数**: >15个独特频道=高竞争，5-15=中等，<5=低竞争
- **时效性**: 近期发布≥70%=高竞争，40-70%=中等，<40%=低竞争

```
Real (现实约束) → Cog (认知模型) → Spec (规约文档) → Work (实际作品)
```

## 常见问题

### Q1: 本项目需要YouTube API密钥吗？
**A**: 不需要！本项目使用MCP fetch进行真实网页浏览，无需申请或配置YouTube API密钥。

### Q2: 如何使用MCP工具收集数据？
**A**: 使用调研工具主程序，会自动生成MCP调用命令：

```bash
# 运行调研工具，会自动提示需要执行的MCP命令
python3 research.py real "Python教程" --regions SG MY TH

# 系统会提示您在Claude Code中执行类似这样的命令：
# @playwright 打开 https://www.youtube.com/results?search_query=Python教程&gl=SG
# @playwright 等待页面加载完成
# @playwright 滚动页面到底部
# @playwright 提取所有视频的标题、频道名、观看量、发布时间
```

**优势**：
- 自动化生成MCP调用命令
- 支持多地区批量收集
- 自动分析竞争度
- 无需API密钥

### Q3: 如何找出竞争少的领域？
**A**: 必须使用真实数据调研，不能预设竞争度：

**步骤1: 收集真实数据**
```bash
# 在Claude Code中调用MCP工具收集数据
@playwright 打开 https://www.youtube.com/results?search_query=Python教程&gl=SG
@playwright 等待页面加载完成
@playwright 滚动页面到底部
@playwright 提取所有视频的标题、频道名、观看量、发布时间
```

**步骤2: 运行真实调研分析**
```bash
python3 examples/real_research.py
```

**系统会基于以下真实指标分析竞争度**：
- **观看量分析**: 平均观看量≥100万=高竞争，10-100万=中等，<10万=低竞争
- **频道数量**: >15个独特频道=高竞争，5-15=中等，<5=低竞争
- **时效性**: 近期发布≥70%=高竞争，40-70%=中等，<40%=低竞争

**重要**: 不同品类、不同国家的竞争度必须基于实际数据动态分析，不能预设！

### Q4: 模式分析结果为空？
**A**:
- 降低 `min_pattern_frequency`（如设为2）
- 增加视频数量
- 检查数据质量和编码
