# 用户系统使用规约 (User Usage Specification)

## 规约概述

本规约定义YouTube视频创作工作流系统的用户操作流程，确保用户能够在2小时内完成从调研到模板的全流程。

## 前置条件

### 系统依赖
- 已完成 `.42cog/real/real.md` 现实约束定义
- 已完成 `.42cog/cog/cog.md` 认知模型构建
- 系统工具链已安装并可正常运行

### 用户能力要求
- 基础内容创作经验
- 了解目标领域基本概念
- 能识别优质内容特征

## 详细规约

### 1. 调研准备阶段

#### 1.1 定义调研目标
**操作步骤**：
1. 明确调研主题（如"老人养生"、"AI工具"）
2. 设定时间限制（建议2小时内完成）
3. 确定期望产出（创作模板数量）

**验收标准**：
- [x] 调研主题明确且具体
- [x] 时间预算合理分配
- [x] 产出目标可量化

#### 1.2 启动工作流
**操作步骤**：
1. 运行系统初始化命令
2. 输入调研主题关键词
3. 系统自动加载预设模板

**验收标准**：
- [x] 工作流正确启动
- [x] 关键词识别成功
- [x] 模板库加载完成

### 2. 事件1：调研分析阶段

#### 2.1 快速数据收集
**操作步骤**：
1. 系统自动搜索相关视频
2. 从1000个候选中筛选
3. 生成10个高质量样本

**验收标准**：
- [x] 数据收集时间 < 30分钟
- [x] 样本质量符合标准
- [x] 去重处理完成

#### 2.2 模式识别
**操作步骤**：
1. 系统分析样本特征
2. 标注共同规律
3. 抽象创作模式

**验收标准**：
- [x] 识别出3-5个核心模式
- [x] 每个模式有明确特征
- [x] 模式可复用验证通过

### 3. 事件2：模式抽象阶段

#### 3.1 生成创作模板
**操作步骤**：
1. 选择最具价值的模式
2. 系统生成对应模板
3. 模板可定制化调整

**验收标准**：
- [x] 模板结构清晰
- [x] 可直接用于创作
- [x] 支持个性化修改

#### 3.2 验证模板可行性
**操作步骤**：
1. 用模板创作示例文案
2. 检查逻辑完整性
3. 优化不完善部分

**验收标准**：
- [x] 示例文案逻辑通顺
- [x] 模板可执行性强
- [x] 优化建议已记录

### 4. 事件3：实战创作阶段

#### 4.1 应用模板创作
**操作步骤**：
1. 选择目标创作主题
2. 套用生成模板
3. 完成实际文案

**验收标准**：
- [x] 文案完整度 > 80%
- [x] 符合平台规范
- [x] 具有独特价值

#### 4.2 效果评估
**操作步骤**：
1. 记录创作时间
2. 评估文案质量
3. 总结改进方向

**验收标准**：
- [x] 创作效率提升明显
- [x] 文案质量达标
- [x] 改进点已明确

### 5. 成果输出阶段

**操作步骤**：
1. 汇总所有调研成果
2. 整理创作模板库

**验收标准**：

## 错误处理

### 常见错误及解决方案

| 错误类型 | 原因分析 | 解决方案 |
|---------|----------|----------|
| 数据收集失败 | 网络问题或关键词过窄 | 检查网络，更换关键词 |
| 模式识别不准确 | 样本质量不高 | 增加样本数量，调整筛选标准 |
| 模板无法使用 | 抽象层级不当 | 回退到更具体的案例分析 |
| 创作效率低 | 模板过于复杂 | 简化模板结构 |
