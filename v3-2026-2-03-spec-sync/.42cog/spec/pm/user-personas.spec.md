# 用户画像规约

> 设计决策的锚点。做功能前先问：这是哪个用户、哪个阶段？

---

## 前台用户

**一句话**：想从数据中发现机会、学习高手的人

### 旅程与需求

| 阶段 | 用户需要 |
|------|----------|
| 全局认识 | 快速了解陌生领域的整体格局；发现套利点和机会；动态跟进变化 |
| 深入学习 | 研究具体高手的模式；理解迭代过程；获取内隐知识 |

### 设计约束

- 先全局后细节，支持逐层下钻
- 可视化优先，突出关键指标和异常值
- 不展示错误，只展示可信结果

---

## 开发者

**一句话**：维护系统、扩展功能的技术人员

### 需求

- 灵活修改和扩展
- 出问题能快速定位
- 新功能易于集成

### 设计约束

- API 优先，日志详尽
- 配置灵活，文档完整
- 报错可以，给完整堆栈

---

## 界面分层

| 界面 | 用户 | 旅程 |
|------|------|------|
| 前台 | 前台用户 | 全局认识 → 深入学习 |
| 后台 | 开发者 | 系统管理、数据调试 |
