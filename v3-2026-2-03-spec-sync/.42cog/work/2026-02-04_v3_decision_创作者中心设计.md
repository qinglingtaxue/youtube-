## 问题背景

### 初期假设（被推翻）

最初设想创作者中心应该提供**"可执行的任务清单"**：

```
创作者中心 = 任务管理系统

结构：
├─ 优先级 ① - 时长优化
│  ├─ [✓ 任务 1] 审查近期 5 个视频的剪辑点
│  ├─ [○ 任务 2] 制作新视频时严控时长在 7-12 分钟
│  └─ [○ 任务 3] 分析用户弃看数据，找出跳跃点
├─ 优先级 ② - 标题优化
└─ 优先级 ③ - 发布频率
```

### 问题发现

用户指出两个致命缺陷：

**问题 1：虚假的可执行性**
- "审查近期 5 个视频的剪辑点" → 系统**无法验证用户是否真的做了**
- 系统无法获取**用户弃看数据**（YouTube Analytics 不提供）
- 任务清单变成了"出题目"而非"提供工具"

**问题 2：无意义的外链**
- 说"点击查看市场规模 Tab" → 用户点过去看到的是**市场数据**
- 但创作者中心说的是**自己应该做什么**
- 两者是**不同的信息层面**，外链根本无法帮助理解

### 根本原因分析

```
错误设计的根本原因：
混淆了两个不同的需求
├─ 需求 A：展示市场机会
│  └─ 由信息报告负责 ✓ 已完成
├─ 需求 B：帮助用户改进自己
│  └─ 原本想用"任务清单"
│  └─ 但系统能力不支持
└─ 结果：虚假承诺 + 用户困惑
```

## 设计决策

### 决策 1：定位改变

**从**：任务管理系统（做什么）
**改为**：诊断式报告（你缺什么，应该改什么）

```
信息报告（市场视角）
├─ 问题："养生"赛道有什么机会？
├─ 答案：市场竞争分散、有内容缺口
└─ 输出：4 个可执行的市场洞察

创作者中心（自身视角）← 这里
├─ 问题：我相对于市场基线差在哪里？
├─ 答案：你的时长偏长、互动率偏低、发布频率偏低
└─ 输出：4 个改进方向 + 可测量的目标
```

**决策理由**：
- ✅ 诚实地承认系统能力边界
- ✅ 提供对用户真正有价值的信息（对标诊断）
- ✅ 避免虚假承诺
- ✅ 用户能自己决策是否改进（赋予控制权）

---

### 决策 2：从 3 个优先级改为 4 个诊断

**移除的概念**：优先级（暗示必须按顺序做）
**引入的概念**：诊断（平行的、可独立决策的）

```
原设计（优先级式）：
优先级 ① - 时长优化（第一个必须做）
优先级 ② - 标题优化（第二个做）
优先级 ③ - 发布频率（第三个做）

新设计（诊断式）：
诊断 ①：时长 vs 市场基线（独立）
诊断 ②：互动率 vs 市场基线（独立）
诊断 ③：发布频率 vs 市场标准（独立）
诊断 ④：选题方向 vs 市场机会（独立）
```

**决策理由**：
- ✅ 用户可以自由选择改进哪个指标
- ✅ 4 个诊断符合工作记忆容量（4±1）
- ✅ 互动率诊断是新增的，来自全局认识数据
- ✅ 避免了"优先级"这种虚假的确定性

---

### 决策 3：诚实标注数据限制

**核心原则**：每条诊断都有"⚠️ 数据限制"段落

```
示例（互动率诊断）：

⚠️ 数据限制：
• 系统无法验证"评论引导"的具体效果
• 互动率受多个因素影响（内容本身、频道知名度等）
• 建议是方向性的，需要用户自己验证
```

**决策理由**：
- ✅ 诚实性 > 虚假承诺
- ✅ 用户能理解"这只是参考，不是保证"
- ✅ 增加用户信任（承认不知道的事）
- ✅ 减少用户后期失望（预期管理）

---

### 决策 4：改进指标必须可测量

**规则**：如果系统无法自动验证，就不作为改进指标

**可测量的指标 ✅**：
```
□ 时长：11 分钟 → 8-10 分钟
  跟踪：YouTube Analytics 的平均视频时长
  自动化：✓ 系统可自动对比

□ 互动率：2.1% → 2.6%
  跟踪：YouTube Analytics 的互动率
  自动化：✓ 系统可自动对比

□ 发布频率：5 天/条 → 3-4 天/条
  跟踪：每月发布视频数
  自动化：✓ 系统可自动对比

□ 选题方向：穴位按摩 11% → 20%
  跟踪：视频元数据中的话题分类
  自动化：✓ 系统可自动对比
```

**不可测量的指标 ❌**：
```
❌ 提高内容质量（定性，无法量化）
❌ 学习 Top 频道风格（定性，无法验证）
❌ 增加评论引导（系统无法检测文案变化）
```

**决策理由**：
- ✅ 下个月能自动对比验证
- ✅ 用户能看到真实的改进结果
- ✅ 避免了虚假承诺的"任务"
- ✅ 建立诚实的反馈循环

---

### 决策 5：从"任务清单"改为"对标建议"

**原设计的虚假任务**：
```
[✓ 任务 1] 审查近期 5 个视频的剪辑点
```

问题：
- 系统无法知道用户是否真的审查了
- 复选框的"完成"是虚假的
- 用户会形成"我完成了任务"的错觉，但实际没改进

**新设计的对标建议**：
```
💡 建议：
• 改进方向：下降到 8-10 分钟范围
• 预期效果：播放量可能 +10-15%
• 执行方式：下次上传视频控制时长

⚠️ 数据限制：
• 系统无法知道用户是否真的执行
• 时长与播放的关系是相关性，非因果性
```

**决策理由**：
- ✅ 提供方向，不虚假承诺验证
- ✅ 用户可自己决定是否改进
- ✅ 诚实地说"系统无法监测"
- ✅ 建立"建议"而非"强制"的关系

---

### 决策 6：外链改为"对标数据"

**原设计（无意义外链）**：
```
📊 相关市场数据 ← 查看市场规模 Tab

用户点过去，看到了市场平均值、频道分布等数据
但这与"我应该改时长到 8-10 分钟"之间没有明显连接
```

**新设计（有意义外链）**：
```
📊 对标数据 ← 查看全局认识·时长分布 Tab

用户需要看：
• 市场上 8-10 分钟视频的平均播放量
• 与自己现在 11 分钟的播放量对比
• 来自全局认识的时长分布数据
```

**决策理由**：
- ✅ 用户点过去，能看到对标的具体数据
- ✅ 外链目的清晰（"看看市场怎么样"）
- ✅ 数据和建议的关联明确
- ✅ 用户能自己验证"为什么要改"

## 设计对标

### 信息报告 vs 创作者中心

| 维度 | 信息报告 | 创作者中心 |
|------|---------|-----------|
| **核心问题** | 市场有什么机会？ | 我应该改什么指标？ |
| **数据来源** | 竞品数据 + Google Trends | 自身数据 vs 市场基线 |
| **输出形式** | 4 个市场结论 + 推理链 | 4 个诊断 + 改进目标 |
| **用户行为** | 学习市场背景 | 对标自己 |
| **决策流向** | 理解"为什么" | 决定"我要改什么" |
| **信息重复** | ❌ 不重复 | ❌ 不重复 |

### 与其他页面的关系

```
用户旅程：

1. 全局认识 → 看市场基础数据
   ├─ 时长分布、互动率基线、频道分布
   └─ 问题：我缺什么？

2. 套利分析 → 看机会排名
   ├─ 有趣度榜、中介中心性榜、程度中心性榜
   └─ 问题：什么话题值得做？

3. 信息报告 → 看市场洞察
   ├─ 4 个结论：市场分散、内容缺口、最优时长、跨语言机会
   └─ 问题：为什么要这样做？

4. 创作者中心 ← 本页面
   ├─ 4 个诊断：你的指标 vs 市场基线
   └─ 问题：我应该改什么？

5. 用户自己 → 执行改进
   ├─ 在创作系统中修改视频时长、选题方向
   └─ 下个月回到创作者中心看对比结果
```

## 数据流向设计

### 4 个诊断的数据来源

| 诊断 | 你的数据 | 市场基线 | 来自信息报告 |
|-----|---------|---------|-------------|
| ① 时长 | YouTube Analytics | 全局认识汇总 | 结论③：4-20min 最优 |
| ② 互动率 | YouTube Analytics | 全局认识汇总 | 结论①：市场概览 |
| ③ 发布频率 | 系统统计 | 市场增长标准 | 结论①：市场成长期 |
| ④ 选题方向 | 视频元数据分析 | 信息报告结论② | 结论②：穴位按摩缺口 |

**设计原则**：
- ✅ 诊断④ 直接来自信息报告结论②（"你应该增加穴位按摩"）
- ✅ 诊断①③ 来自市场分析（平均数据）
- ✅ 诊断② 来自全局认识（基础数据）
- ✅ 避免了任何虚造数据或假设

## 实现约束

### UI 设计检查清单

所有设计都遵循以下约束（保存在 `real.md Appendix E`）：

**职责边界**：
- ✅ 只做诊断，不做任务清单
- ✅ 不重复展示市场数据
- ✅ 每条诊断都有数据限制标注
- ❌ 禁止虚假的"任务完成"复选框

**数据可信度**：
- ✅ 你的数据 = YouTube Analytics（官方）
- ✅ 市场基线 = 2,340 条视频 + 186 频道
- ✅ 改进预期 = 基于市场统计的估算（±5-10%）
- ❌ 禁止承诺"保证"或"一定会"

**改进指标**：
- ✅ 每个指标都可测量（具体数字）
- ✅ 下个月能自动对比验证
- ✅ 系统会展示"实现情况"
- ❌ 禁止定性指标（如"内容质量"）

## 决策优势

### 为什么这样设计更好？

**1. 诚实 > 虚假**
```
原设计：[✓ 任务 1] 审查视频 → 虚假完成感
新设计：建议改为 8-10min → 用户自己验证
```

**2. 参考 > 强制**
```
原设计：你必须改进频率到 3-4 天/条 → 可能无法做到
新设计：建议改为 3-4 天/条（基于你的制作能力） → 用户自己决策
```

**3. 诊断 > 任务**
```
原设计：任务清单（系统无法验证）
新设计：诊断卡片（用户可自我验证）
```

**4. 对标 > 指导**
```
原设计：外链到市场数据（无关连）
新设计：对标建议（用户能理解"为什么改"）
```

**5. 下月对比 > 今月承诺**
```
原设计：系统说"改了就能增长" → 虚假承诺
新设计：下个月自动对比 → 真实的反馈循环
```

## 风险与缓解

### 风险 1：用户期望过高

**风险**：用户看到建议后，以为一定会改进
**缓解**：在每条诊断都标注"⚠️ 数据限制"和"±YY% 误差范围"

### 风险 2：用户不知道应该改什么

**风险**：4 个诊断都有，用户不知道优先级
**缓解**：
- 每个诊断都清晰说明"预期收益"
- 用户可自由选择改哪个（赋予控制权）
- 建议基于市场数据（有据可查）

### 风险 3：用户发现系统"骗"他

**风险**：下个月对比后，改进效果不如预期
**缓解**：
- 提前说明"这是估算，实际可能不同"
- 标注影响因素（内容本身、频道知名度等）
- 邀请用户自己分析"为什么没达到"

### 风险 4：数据无法自动获取

**风险**：无法自动对比"选题方向"（需要人工分类）
**缓解**：
- 使用视频元数据中的话题关键词统计
- 如果无法精确，显示"基于关键词统计"
- 透明说明"可能有偏差"
