## 决策背景

在设计套利分析页面布局时，经过多轮迭代和用户反馈，形成了当前的设计方案。本文档记录关键决策及其原因。

**页面定位**：从首页「三大功能入口」进入的核心功能页，整合视频榜单和频道发现，帮助用户发现「被低估的创作机会」。

**核心公式**：
```
有趣度 = 中介中心性 ÷ 程度中心性
       = 桥梁能力 ÷ 热门程度
       = 价值被低估的程度
```

---

## 核心设计原则

### 原则 1：按用户决策流程分组

**问题**：初版设计有 7 个平铺的 Tab（视频榜单、频道发现、时长分布、趋势套利、话题套利、中心性分析、综合建议），超出人类工作记忆舒适范围（4±1 个）

**用户反馈**：
> "根据人类的工作记忆，一下子看 7 个（Tab 1-7）太多了，你分一下组"

**决策**：按「看数据 → 找机会 → 去行动」分为 2 个组 + 1 个外链

```
┌─ 📊 看数据 ─────────────┐  ┌─ 💰 找机会 ─────────────┐
│ 了解市场现状            │  │ 发现套利空间            │
│                         │  │                         │
│ • 视频榜单              │  │ • 有趣度榜（套利机会）  │
│ • 频道发现              │  │ • 中介中心性榜（桥梁）  │
│ • 赛道分析              │  │ • 程度中心性榜（热门）  │
│ • Google 趋势           │  │                         │
└─────────────────────────┘  └─────────────────────────┘

🎯 去行动 → 在洞察系统「信息报告」Tab 中查看综合建议
```

**好处**：
- 用户一眼只需处理 2-3 个选项
- 决策流程清晰：先看数据，再找机会，最后行动

---

### 原则 2：三个中心性榜单独立展示

**问题**：初版设计把中心性分析放在一个 Tab 内，用户需要切换才能看到不同榜单

**用户反馈**：
> "程度中心性分析呢？有趣度分析呢？重新分组"

**决策**：在「找机会」组中提供三个独立榜单，让用户直接看到有趣度公式的三个组成部分

| 榜单 | 含义 | 用户价值 |
|------|------|---------|
| 💎 有趣度榜 | 价值高但知道的人少 | 套利机会排名 |
| 🌉 中介中心性榜 | 能连接多个群体的桥梁话题 | 找跨领域机会 |
| 🔥 程度中心性榜 | 已经被广泛讨论的热门话题 | 了解竞争程度 |

**四象限解读**（帮助用户理解三个榜单的关系）：
```
    高程度中心性
         ↑
 📉 红海区  │  🎯 竞争激烈
 (避开)    │  (大V适合)
───────────┼───────────→ 高中介中心性
 ❄️ 冷门区  │  💎 套利区
 (不值得做) │  (新手机会)
         ↓
    低程度中心性
```

---

### 原则 3：排序控制与榜单类型并排

**问题**：初版设计的排序设置独立成块，占用首屏空间；用户想方便地筛选不同时间条件

**用户反馈**：
> "方便用户筛选，查看不同时间条件下的榜单"
>
> "当前: 7天内发布的视频，按播放量从高到低排序 [▶ 展开筛选条件] 放在排序里面"

**决策**：左侧榜单类型，右侧排序控制，同一行展示

```
┌─ 榜单类型 ──────────────────────────────┬─ 排序 ───────────────────────┐
│ ● 🦄 黑马视频（小频道高播放） ← 默认    │ 时间: [7天内 ▼]              │
│ ○ 🔥 爆款视频（播放量最高）             │ 字段: [播放量 ▼]             │
│ ○ 📈 增长最快（日均增长）               │ 方向: [高→低 ▼]              │
│ ○ 💬 高互动（点赞+评论最高）            ├──────────────────────────────┤
│                                         │ 7天内，按播放量高→低排序     │
│                                         │             [▶ 展开筛选条件] │
└─────────────────────────────────────────┴──────────────────────────────┘
```

**好处**：
- 首屏更紧凑
- 用户同时看到「看什么」和「怎么排」
- 自然语言显示当前规则

---

### 原则 4：用赛道分析替换时长分布

**问题**：初版设计包含「时长分布」Tab，但价值不够高

**用户反馈**：
> "时长分布没那么重要，还不如哪个类型的赛道更值得关注？"

**决策**：
- ❌ 删除「时长分布」Tab
- ✅ 新增「赛道分析」Tab（在「看数据」组中）

**原因**：
- 时长分布是静态数据，用户看一次就够了
- 赛道分析能帮用户决定「进入哪个细分领域」，更有决策价值

---

### 原则 5：综合建议移到信息报告

**问题**：初版设计把「综合建议」和「行动指南」放在套利分析页面内

**用户反馈**：
> "🎯 去行动（组 3）← 获取行动指南，放在"信息报告"里面"

**决策**：
- ❌ 不在套利分析页面放「综合建议」Tab
- ✅ 用外链指向洞察系统「信息报告」Tab

**原因**：
- 套利分析专注于「数据和机会发现」
- 行动建议属于「决策输出」，应该在信息报告中呈现
- 避免页面功能过于臃肿

---

### 原则 6：新增 Google 趋势入口

**问题**：初版设计缺少外部数据源的入口

**用户反馈**：
> "数据榜单 💰 套利发现 🎯 行动 加一个 google 趋势榜单"

**决策**：在「看数据」组中新增「Google 趋势」入口

**原因**：
- Google 搜索趋势是发现新兴话题的重要信号
- 与 YouTube 数据形成互补（搜索需求 vs 内容供给）
- 帮助用户发现「Google 上涨但 YouTube 还少」的内容缺口

---

### 原则 7：专业术语必须解释

**问题**：有趣度、中介中心性、程度中心性、趋势比等都是专业术语，用户看不懂

**决策**：每个专业术语都需要翻译和解释

| 术语 | 解释方式 |
|------|---------|
| 有趣度 | 括号说明「价值÷传播」+ 进度条可视化 + 四象限图 |
| 中介中心性 | 翻译为「桥梁能力」+ 显示连接的群体 |
| 程度中心性 | 翻译为「热门程度」+ 显示竞争程度标签 |
| 趋势比 | 翻译为「近期÷历史频率」+ 百分比变化 |
| 效率分数 | 括号说明「平均播放÷订阅」 |

**参考**：详见 cog.md 用户认知模型，用户不理解的概念需要特殊处理

---

### 原则 8：默认选中套利价值最高的选项

**问题**：每个 Tab 有多个子选项，需要决定默认选中哪个

**决策**：默认选中套利价值最高的选项

| Tab | 默认选中 | 原因 |
|-----|---------|------|
| 视频榜单 | 🦄 黑马视频 | 小频道高播放 = 内容本身有价值 |
| 频道发现 | 🦄 黑马频道 | 低订阅高播放 = 值得模仿 |
| 找机会 | 💎 有趣度榜 | 这是用户最关心的套利机会 |
| 趋势套利 | 🚀 上升话题 | 趋势比 > 1.5 的话题有红利 |

**原因**：符合页面定位「发现被低估的创作机会」

## 功能清单

### 📊 看数据（组 1）

| 功能 | 用户问题 | 核心内容 |
|------|---------|---------|
| 视频榜单 | 哪些视频值得模仿？ | 黑马/爆款/增长/高互动榜 |
| 频道发现 | 哪些频道值得关注？ | 黑马/高效/头部/新兴频道 |
| 赛道分析 | 哪个赛道值得进入？ | 赛道竞争度、机会分析 |
| Google 趋势 | 什么话题正在上升？ | 搜索热度变化 |

### 💰 找机会（组 2）

| 功能 | 用户问题 | 核心内容 |
|------|---------|---------|
| 有趣度榜 | 哪里有套利空间？ | 高价值低传播的话题/频道/标题词 |
| 中介中心性榜 | 哪些话题能连接多群体？ | 桥梁话题，跨领域机会 |
| 程度中心性榜 | 哪些话题竞争激烈？ | 热门话题，需要避开或找细分 |

### 🎯 去行动（外链到信息报告）

- 综合建议（根据创作者阶段定制）
- 行动清单（可导出的 Todo List）

## 数据来源与约束

| 数据项 | 来源 | 约束 |
|-------|------|------|
| 视频基础数据 | yt-dlp Stage1 | 标题、播放量、时长 |
| 视频详情 | yt-dlp Stage2 | 点赞、评论、发布日期（仅日期级）|

| 时长分类 | duration 字段 | 必须用 YouTube 原生分类 |

## 图表选择决策

> 通用决策详见：`2026-02-04_v3_decision_图表选择策略.md`
>
> 页面定位对比见：第 1 节「页面定位与图表策略」

### 页面定位与图表策略

**套利分析**：找到机会（怎么做）— 直观理解数据在信息网络中的分布

| 维度 | 本页面策略 |
|------|-----------|
| **核心问题** | 哪里有机会？ |
| **用户目标** | 发现套利空间 |
| **数据关系** | 网络、关联、中心性 |
| **图表策略** | **聚焦网络图**理解中心性 |
| **多样性目标** | 4/8 类型，深度优先 |

### 为什么套利分析页面聚焦网络图？

> 完整认知映射表见：`2026-02-04_v3_decision_图表选择策略.md` 第 3 节
>
> **用户反馈**：「信息网络图可以帮助直观的理解中介中心性、程度中心性、有趣度」

套利分析的核心是**有趣度公式**：
```
有趣度 = 中介中心性 ÷ 程度中心性
       = 桥梁能力 ÷ 热门程度
       = 价值被低估的程度
```

这三个指标都是**网络属性**，用户无法通过数字直接理解（如「中介中心性=0.85」毫无意义）。

**网络图的认知映射**：

| 抽象概念 | 可视化方式 | 用户直观理解 |
|---------|-----------|-------------|
| 中介中心性 | 节点大小 | 节点越大 = 越是信息桥梁 |
| 程度中心性 | 连线数量 | 连线越多 = 传播越广 |
| 有趣度 | 金色高亮 | 金色 = 套利机会 |
| 内容相似度 | 节点距离 | 越近 = 越相似 |

### 本页面图表选择（细化应用）

> 决策依据见：`2026-02-04_v3_decision_图表选择策略.md` 第 2 节决策矩阵

| 数据需求 | 数据类型 | 数量级 | 选择的图表 | 决策依据 |
|---------|---------|-------|-----------|---------|
| 展示有趣度公式 | 公式解释 | 1个 | **公式卡片** | 核心概念必须可视化解释 |
| 信息网络关系 | 网络关系 | 50-200节点 | **力导向网络图** | 网络关系 50-200 → 力导向图+聚类 |
| 中心性组合分布 | 两指标相关 | 100+点 | **散点图** | 两指标相关 50-500 → 散点图 |
| 有趣度榜单 | 排名数据 | 20条 | **表格+进度条** | 排名 10-50 → 表格+迷你条 |
| 中介中心性榜单 | 排名数据 | 20条 | **表格+进度条** | 排名 10-50 → 表格+迷你条 |
| 程度中心性榜单 | 排名数据 | 20条 | **表格+进度条** | 排名 10-50 → 表格+迷你条 |
| 四象限定位 | 两指标相关 | 1个 | **坐标系解释图** | 帮助理解三榜单关系 |
| 关键词分布 | 文本数据 | 50-100词 | **词云** | 关键词 20-200 → 词云 |

### 多样性检查

> 检查清单见：`2026-02-04_v3_decision_图表选择策略.md` 第 4 节

```
✅ 数字展示类：公式卡片、有趣度分数卡片
✅ 关系类（核心）：网络图（有趣度网络）、散点图（中心性分布）
✅ 文本类：榜单表格（有趣度/中介/程度三榜单）
✅ 分布类：词云（关键词分布）
⚠️ 比例类：可选添加（有趣度分布饼图）
⚠️ 比较类：可选添加（中心性对比柱状图）
```

**多样性评估**：4/8 类型覆盖

**设计说明**：套利分析页面不追求图表多样性最大化，而是**聚焦网络图**帮助用户理解中心性概念。这是因为：
1. 页面定位是「找机会」，不是「看全貌」
2. 有趣度公式的三个组成部分都是网络属性
3. 网络图是理解中心性的最佳（也是唯一直观的）工具

### 网络图设计规范

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   控制面板:                                                      │
│   ┌────────────────┐ ┌────────────────┐ ┌────────────────┐      │
│   │ 节点大小:      │ │ 节点颜色:      │ │ 显示节点:      │      │
│   │ [有趣度 ▼]     │ │ [有趣度 ▼]     │ │ [全部 ▼]       │      │
│   └────────────────┘ └────────────────┘ └────────────────┘      │
│                                                                 │
│   网络图区域:                                                    │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │      ●───────●                     节点大小 = 指标值     │   │
│   │     /│       │\                    金色 = 高有趣度      │   │
│   │    ● │   ◉   │ ●                   紫色 = 高中介        │   │
│   │     \│  (大) │/                    红色 = 高程度        │   │
│   │      ●───────●                                          │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   图例:                                                          │
│   [●金] 高有趣度 [●紫] 高中介 [●红] 高程度 [●灰] 低中心性       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**交互设计**：
- 悬停：显示节点详情（视频标题、有趣度、中介、程度）
- 点击：高亮关联节点，显示详情面板
- 切换：可切换节点大小/颜色的映射指标
- 缩放：滚轮缩放，拖拽平移

## 决策总结

| 决策 | 原因 |
|------|------|
| 功能分为 2 组 + 1 外链 | 符合工作记忆 4±1 原则 |
| 三个中心性榜单独立展示 | 帮助用户理解有趣度公式的组成 |
| 排序控制与榜单类型并排 | 首屏更紧凑，信息密度更高 |
| 用赛道分析替换时长分布 | 赛道选择比时长分布更有决策价值 |
| 综合建议移到信息报告 Tab | 套利分析专注数据和机会发现 |
| 新增 Google 趋势入口 | 与 YouTube 数据形成互补 |
| 专业术语都有翻译和解释 | 降低认知门槛 |
| 默认选中套利价值最高的选项 | 符合页面定位 |
| 四象限图解释中心性组合 | 直观展示套利区在哪里 |
| **聚焦网络图展示中心性** | 网络图是理解中心性的最佳工具 |
| **添加词云展示关键词** | 帮助用户快速识别热门话题 |
