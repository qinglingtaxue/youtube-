## 布局关键特征

### ✅ 遵循的原则

| 原则 | 体现 |
|------|------|
| **单页推理链** | 假设 → 4 条结论 → 综合结论 → 行动，一气呵成 |
| **结论顺序优化** | ① 市场机会 ② 选题指导 ③ 时长指导 ④ 拓宽思路 |
| **展开设计** | 每条结论默认折叠，点击「▼ 展开数据」查看推理链 + 嵌入图表 |
| **视觉层级** | 结论标题 + 摘要始终可见，推理细节在展开区隐藏 |
| **数据引用不跳转** | 图表在页面内嵌入（来自全局认识/套利分析），无外链 |
| **置信度透明** | 每条结论都显示置信度（75-88%） |

### 📊 结论排序逻辑

| 位置 | 结论 | 用户问题 | 作用 |
|------|------|---------|------|
| 1️⃣ | 市场竞争分散 | "竞争激烈吗？" | 鼓励用户有信心切入 |
| 2️⃣ | 内容缺口 | "什么话题最值得做？" | 指导具体选题 |
| 3️⃣ | 最优时长 | "什么时长最高效？" | 指导内容规划 |
| 4️⃣ | 跨语言机会 | "如何拓宽思路？" | 启发多元视角 |

### 🔄 与原 5 结论版本的差异

| 项目 | 原版本（5 结论） | 新版本（4 结论） |
|------|-----------------|-----------------|
| **删除的结论** | ❌ 长标题 (30+ 字) 播放量 4 倍 | ✅ 移除（低可行性） |
| **时长结论位置** | 第 5 位（最后） | 第 3 位（中间） |
| **综合结论** | 提及长标题策略 | 强调时长 + 跨语言 |
| **总页面高度** | 较长（5 个卡片） | 中等（4 个卡片） |

### 💡 设计原理：为什么这样对应？

| 原理 | 说明 |
|------|------|
| **套利分析是数据** | 套利分析展示「事实」（什么话题有趣度高、什么话题竞争少） |
| **信息报告是洞察** | 信息报告展示「含义」（为什么这个话题值得做，如何做） |
| **有趣度公式的应用** | 有趣度 = 中介中心性 ÷ 程度中心性 = 价值 ÷ 传播<br>结论② 正是有趣度公式在实战中的体现 |
| **跨页面数据引用** | 信息报告不重复数据，而是「引用+解释」套利分析的结果 |
| **决策链完整性** | 用户在套利分析看到机会 → 在信息报告理解原因 → 在行动中心执行 |

### 🎯 用户在两个页面间的信息获取

**用户在套利分析中获得**：
- ✅ 有趣度排名（Top 10 话题/频道/标题词）
- ✅ 中心性对比（谁是桥梁，谁是热门）
- ✅ 趋势变化（Google 搜索热度）
- ❌ 为什么该选这个话题（缺乏推理链）

**用户在信息报告中获得**：
- ✅ 完整推理链（假设 → 数据 → 结论）
- ✅ 综合决策建议（市场 + 选题 + 时长 + 跨语言）
- ✅ 嵌入图表（不需跳转，完整沉浸）
- ❌ 所有话题的详细排名（那是套利分析的职责）

### 全局时间范围筛选器（全局通用状态）

✅ **设计原则**：时间范围是**全局状态**，用户在任何页面改时间，所有页面都同步更新

```
用户改变时间范围（上方全局筛选器）
  ↓
全局状态更新：globalState.timeRange = '过去 7 天'
  ↓
所有页面同步刷新数据：
  ├─ 全局认识：显示 7 天内的数据
  ├─ 套利分析：显示 7 天内的数据
  ├─ 信息报告：结论数据根据 7 天内的统计更新
  └─ 创作者行动中心：推荐根据 7 天内的数据调整
```

- 全局状态：`globalState = { keyword: '养生', timeRange: '过去 3 个月' }`
- 信息报告读取全局状态，展示当前参数
- 结论数据：根据 `globalState.timeRange` 动态查询，显示该时间范围内的分析
- 同步机制：用户在任何页面改时间范围 → 全局状态更新 → 所有页面（包括信息报告）重新渲染
