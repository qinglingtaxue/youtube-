> 相关文档: [2026-02-04_v3_design_信息报告页面.md](../spec/2026-02-04_v3_design_信息报告页面.md)

## 核心决策

### 决策 1: 信息报告改为「推理链式报告」而非「Dashboard 多 Tab」

**原始设计**: 4 个独立 Tab（分析摘要 / 机会评估 / 行动清单 / 趋势预警）

**变更设计**: 单页推理链式报告

#### 决策原因

| 维度 | 原设计问题 | 新设计优势 |
|------|---------|---------|
| **信息关系** | Tab 间缺乏逻辑串联，用户需自行拼凑 | 假设 → 结论 → 综合结论，推理清晰 |
| **页面定位** | 混合了多个不同职责（数据展示+行动建议） | 纯粹的分析结论汇总（去行动的入口） |
| **用户心智** | "看了很多 Tab，不知道总结是什么" | "读完就知道该做什么" |
| **重复展示** | Tab 2 的机会排名重复了套利分析页 | 不重复数据，只汇总跨页结论 |
| **认知流** | Tab 间切换中断阅读连贯性 | 单页滚动，沉浸式阅读体验 |

#### 设计框架

```
假设（研究问题）
  ↓
结论 1~N（引用全局认识/套利分析数据，点击展开推理链）
  ↓
综合结论（汇总回答假设）
  ↓
行动入口（导出 + 外链）
```

---

### 决策 2: 删除「标题长度」相关的结论

**原始设计**: 包含"长标题 (30+字) 播放量是短标题的 4 倍"作为结论 4

**变更设计**: 删除该结论，保留 4 条高优先级结论

#### 决策原因

| 指标 | 评估 | 说明 |
|------|------|------|
| **可行动性** | ❌ 低 | 创作者写视频标题时必然会写长标题，无需特别提示 |
| **优先级** | ❌ 低 | 不是选择话题时的决策因素，是执行层细节 |
| **数据价值** | ❌ 低 | 标题字符数分析不如话题缺口、时长选择重要 |
| **信息报告职责** | ❌ 不符 | 信息报告是"去行动的指南"，不是"执行手册" |

#### 精简后的 4 条结论

1. **市场竞争分散，新人有机会** — 鼓励切入
2. **「穴位按摩」等话题内容缺口** — 指导选题
3. **4-20min 中视频最优** — 指导时长选择
4. **Tai Chi 跨语言机会** — 拓宽思路

---

### 决策 3: 删除「行动清单」独立页面

**原始设计**: 页面 B 为"行动清单（进度追踪 + P0/P1/P2 勾选 + 导出）"

**变更设计**: 删除行动清单页面，仅保留信息报告单页

#### 决策原因

| 维度 | 分析 |
|------|------|
| **使用频率** | 行动清单是一次性导出，不需要反复交互 |
| **功能重复** | 创作者行动中心已承载个性化行动管理 |
| **页面职责分离** | 信息报告 = 分析结论；行动管理 = 创作者行动中心 |
| **用户流程** | 读报告 → 决定是否执行 → 去行动中心执行，清晰的职责边界 |

#### 替代方案

- 信息报告底部保留两个按钮：
  - `📥 导出报告 (Markdown)` — 用户自留备案
  - `🎯 创作者行动中心 →` — 跳转个性化工具

---

### 决策 4: 删除「创作者阶段选择器」

**原始设计**: 机会评估 Tab 中有"新手 / 成长期 / 成熟期"选择器

**变更设计**: 删除选择器，个性化推荐转移至创作者行动中心

#### 决策原因

| 维度 | 说明 |
|------|------|
| **定位冲突** | 信息报告是**客观的分析汇总**，不提供个性化定制 |
| **功能归属** | 个性化推荐（按创作者阶段）属于**创作者行动中心** |
| **职责分离** | 信息报告只展示共通性洞察，避免混入个性化逻辑 |
| **页面复杂度** | 移除选择器后页面聚焦，阅读更清晰 |

---

### 决策 5: 图表不跳转，在报告内嵌入

**原始设计**: 点「引用数据源」→ 跳转到全局认识/套利分析页面

**变更设计**: 结论卡片展开时，直接嵌入引用的图表（不跳转）

#### 决策原因

| 维度 | 原设计问题 | 新设计优势 |
|------|---------|---------|
| **沉浸度** | 跳转打破阅读流 | 在页面内查看，保持沉浸 |
| **认知负荷** | 用户需记住页面间关系 | 图表就在眼前，一目了然 |
| **返回成本** | 用户需返回报告，找回阅读位置 | 无需返回，连贯阅读 |
| **设计理念** | Tab 式切割注意力 | 单页报告，一气呵成 |

#### 实现方式

```
结论卡片
  ├─ 标题 + 摘要（始终可见）
  ├─ 数据来源标签（始终可见）
  └─ ▶ 展开数据
      ├─ 推理过程（步骤 → 步骤 → 结论）
      └─ 嵌入图表（来自全局认识/套利分析的截图或重新渲染）
```

用户无需离开页面即可理解整个推理链。

## 与其他页面的关系

### 跨页面数据依赖

```
全局认识页面
  ├─ 市场规模 Tab → 结论①数据
  ├─ 内容分布 Tab → 结论③数据
  └─ 频道格局 Tab → 结论①数据

套利分析页面
  ├─ 有趣度榜单 → 结论②数据
  └─ 跨语言机会 → 结论④数据

Google Trends（外部）
  ├─ 穴位按摩 +76% → 结论②
  └─ Tai Chi +533% → 结论④

创作者行动中心（外链）
  └─ 信息报告 → "去行动中心执行"按钮
```

### 数据嵌入策略

| 结论 | 数据源 | 嵌入图表 | 备注 |
|------|--------|---------|------|
| ① 市场竞争 | 全局认识 | 频道集中度柱状图 + 增长趋势折线图 | 来自「市场规模」「频道格局」Tab |
| ② 内容缺口 | Google Trends + 套利分析 | 趋势对比表（GT vs YT视频数） | Google Trends 外链说明 |
| ③ 最优时长 | 全局认识 | 时长分布环形图 + 平均播放柱状图 | 来自「内容分布」Tab |
| ④ 跨语言机会 | Google Trends + 套利分析 | 趋势爆发曲线 + 市场空白对比 | Google Trends 外链说明 |

## 设计约束遵循

### CLAUDE.md 规范

| 约束 | 检查项 | 遵循状态 |
|------|--------|---------|
| 工作记忆 ≤5 | 不使用 Tab（单页） | ✅ |
| 用户能理解的指标 | 使用播放量、趋势比、话题名 | ✅ |
| 数据和行动分离 | 信息报告仅汇总结论，行动建议外链 | ✅ |
| 禁止模糊概念 | 无"综合排序""热门"等模糊词汇 | ✅ |
| 视觉层级 | 推理链从上到下：假设 → 结论 → 综合 | ✅ |
| 首屏内容优先级 | 首屏显示假设 + 第一条结论 | ✅ |
