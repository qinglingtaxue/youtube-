# 设计自我审查快速参考

> **何时用**：完成 design 文件后，发送给用户前
>
> **目标**：检查完成 ✅ 所有项，确保一次通过（0 反馈）

---

## ⚡ 5 分钟快速检查

### 1️⃣ Tab 设计（如果页面有 Tab）
```
🔍 快速问：我的页面有超过 5 个 Tab 吗？

是 → 分组检查
   □ 按"看数据→找机会→去行动"分组了吗？
   □ 每组 ≤ 4 个 Tab？
   □ 总分组数 ≤ 3？

否 → 跳过分组检查
   □ Tab 名称清晰易懂吗？（避免"其他"、"更多"）
   □ 数据展示和行动分开了吗？
```

### 2️⃣ 视觉层级
```
🎯 快速问：首屏第一眼看到什么？

理想：用户的核心操作入口（如搜索框）
     ↓
     用户历史（搜索记录）
     ↓
     核心功能入口（≤3 个）
     ↓
     统计数据/概览（页面最底部）

检查清单：
□ 首屏第一眼是核心操作吗？
□ 统计数据在页面底部吗？
□ 功能入口 ≤ 3 个吗？
```

### 3️⃣ 排序/筛选（如果页面有）
```
🎛️ 快速问：排序设计是否拆分为 3 个独立选择？

标准格式：
时间范围: [7天内 ▼]  |  排序字段: [播放量 ▼]  |  方向: [高→低 ▼]

当前: 7天内发布的视频，按播放量从高到低排序

检查清单：
□ 是否拆分为 3 个（时间 + 字段 + 方向）？
□ 是否避免了"综合排序"等模糊概念？
□ 是否用自然语言显示当前规则？
□ 筛选条件默认折叠吗？
□ 筛选都能用自然语言解释吗？
```

### 4️⃣ 用户能否理解？
```
🧠 快速问：我的页面有这些术语吗？

❌ 禁止用（用户不懂）：
  - 互动率
  - 效率分数
  - 中心性 / 中介中心性 / 程度中心性
  - P 分位数
  - 置信度
  - 综合排序 / 热门 / 推荐

✅ 用户能理解：
  - 播放量、点赞、评论、订阅
  - 时长、发布时间
  - 频道名称、视频标题

检查清单：
□ 是否避免了上述黑名单术语？
□ 复杂概念是否有括号解释？
□ 是否用了 YouTube 不支持的时长分档？
   - ❌ 自定义：0-5分、1-3分、30分+
   - ✅ YouTube 原生：<4分、4-20分、>20分
```

### 5️⃣ 页面职责边界
```
🏗️ 快速问：这个页面的职责是什么？

检查清单：
□ 页面承担的用户任务 ≤ 2 种吗？
□ 数据展示和行动建议分开了吗？
   - 如果混杂 → 拆分为两个页面或 Tab
□ 是否有明确的"出口"（导出/外链）？
   - 如果数据页 → 有外链到"行动页"吗？
   - 如果分析页 → 有外链到"执行页"吗？
```

## 🎬 使用流程

### 步骤 1：完成 design 文件
画完布局草稿和交互说明

### 步骤 2：打开这份快速参考
选择"⚡ 5 分钟快速检查"或"📋 完整清单"

### 步骤 3：逐项检查
☐ → ✅（调整设计或记录原因）

### 步骤 4：记录审查结果
在 design 文件顶部添加：
```
## 自我审查状态：✅ 全部通过

已检查项：A/B/C/D/E/F/G
需调整项：C1, D2（见下文）
```

### 步骤 5：发给用户
附上简短说明："已完成自我审查，调整项：#C1（排序拆分）, #D2（术语修改）"

## 💡 常见陷阱

### 陷阱 1：Tab 太多
❌ 5 个 Tab 不分组 → 用户选择困难
✅ 6+ 个 Tab 必须分组（每组 ≤ 4）

### 陷阱 2：首屏放了统计数据
❌ 页面一打开就是"总播放量：123万"
✅ 统计数据应该在页面底部

### 陷阱 3：排序只有一个下拉框
❌ 排序: [综合排序 ▼]  → 什么是"综合"？
✅ 时间: [7天内 ▼] | 字段: [播放量 ▼] | 方向: [高→低 ▼]

### 陷阱 4：用"互动率"作为筛选条件
❌ "筛选互动率 > 5%"  → 大多数用户不知道这是什么
✅ 用"用户参与高"，并括号说明"点赞+评论÷播放量 > 5%"

### 陷阱 5：混杂数据和行动
❌ 同一个页面既展示数据，又提供"点击执行任务"
✅ 数据页展示数据 + 外链到"行动页"

### 陷阱 6：自定义时长分档
❌ 0-5分、5-10分、10-30分  → YouTube 不支持
✅ <4分、4-20分、>20分  → YouTube 原生分档

---

## 🚨 红线（触犯直接不过）

这些问题不能"记录原因"跳过，必须在第 2 阶段改掉：

1. **Tab > 5 个且不分组**
2. **首屏第一眼是统计数据**
3. **排序不是 3 个独立选择**
4. **页面混杂"数据展示"和"行动建议"**
5. **筛选条件使用用户不懂的术语**
6. **时长用自定义分档（不是 YouTube 原生）**
