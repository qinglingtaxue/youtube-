## 模式2：订阅效率分析

### 方法

1. **订阅分档**：
   - 1M+ 大V
   - 100K-1M 中V
   - 10K-100K 小V
   - 1K-10K 新人
   - <1K 素人
2. **效率计算**：`效率 = AVG(view_count) / AVG(subscriber_count)`
3. **对比**：不同档次效率差异

### 偏差风险

- 素人样本可能存在已删除/私有视频

### 置信度判定

**高**（N=956频道）

## 模式3：时长最优区间分析

### 方法

1. **时长分档**：
   - <1分钟（短）
   - 1-3分钟（短中）
   - 3-10分钟（中）
   - 10-30分钟（中长）
   - 30分+（长）
2. **指标计算**：
   - 平均播放：`AVG(view_count)`
   - 点赞率：`AVG(like_count / view_count * 100)`
3. **排序**：按平均播放降序

### 偏差风险

- 不同内容类型的最优时长可能不同

### 置信度判定

**高**（N=2340）

## 模式4：内容类型分析

### 方法

1. **类型识别**：标题关键词匹配
   - 功法教学：含"八段锦"/"太极"/"气功"
   - 饮食养生：含"食疗"/"饮食"/"吃"
   - 穴位按摩：含"穴位"/"按摩"
   - 健康警示：含"警告"/"危险"/"千万别"
   - 等等
2. **聚合统计**：按类型分组，计算均播放、时长、点赞率

### 偏差风险

- 基于标题关键词匹配，可能误分类
- 一个视频可能属于多个类型

### 置信度判定

**中**（分类方法有局限）

## 模式7-10：标题特征分析

### 方法

#### 模式7：数字效果

```sql
CASE WHEN title REGEXP '[0-9]' THEN '含数字' ELSE '不含数字' END
```

#### 模式8：感叹号效果

```sql
CASE WHEN title LIKE '%！%' OR title LIKE '%!%' THEN '有感叹号' ELSE '无感叹号' END
```

#### 模式9：Hashtag效果

```sql
CASE WHEN title LIKE '%#%' THEN '有Hashtag' ELSE '无Hashtag' END
```

#### 模式10：句型效果

```sql
CASE WHEN title LIKE '%？%' OR title LIKE '%?%' OR title LIKE '%如何%' OR title LIKE '%怎么%'
     THEN '问句/How-to' ELSE '陈述句' END
```

### 标题长度分析

```sql
CASE
    WHEN CHAR_LENGTH(title) >= 50 THEN '超长(50+字)'
    WHEN CHAR_LENGTH(title) >= 30 THEN '长(30-50字)'
    WHEN CHAR_LENGTH(title) >= 15 THEN '中(15-30字)'
    ELSE '短(<15字)'
END as title_length
```

### 偏差风险

- 相关性非因果，可能与内容类型、创作者水平混淆
- 标题特征可能互相交叉影响

### 置信度判定

**中**：相关性非因果

## 模式23：话题垄断度分析

### 方法

1. **话题分组**：按话题关键词分组
2. **频道排名**：每个话题内按频道总播放降序排名
3. **Top3份额**：`Top3播放 / 总播放`
4. **垄断分类**：
   - >90%：极度垄断
   - 80-90%：高度垄断
   - 70-80%：较垄断
   - <70%：竞争市场

### 偏差风险

- 无重大偏差

### 置信度判定

**高**（无偏差）
