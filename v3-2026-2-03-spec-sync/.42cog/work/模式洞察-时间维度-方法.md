## 模式5：周末发布效果分析

### 方法

1. **时间提取**：从 `timestamp` 提取星期几（1=周日, 7=周六）
2. **分组聚合**：按星期分组，计算 AVG(view_count)、MAX(view_count)
3. **排序对比**：按平均播放量降序排列

### 偏差风险

- 可能受单个爆款视频影响（如周日有1905万的爆款）
- 需排除异常值后重新计算验证

### 置信度判定

**低→中**：样本量足够，但异常值影响待排除

## 模式13：话题生命周期分析

### 方法

1. **话题识别**：标题关键词匹配（如"八段锦"、"太极"、"食疗"等）
2. **生命周期计算**：
   - 首次出现：`MIN(published_at)`
   - 活跃跨度：`MAX(published_at) - MIN(published_at)`
3. **热度分类**：
   - 短周期型（2-4个月）：人物/事件驱动
   - 中周期型（6-12个月）：健康焦虑驱动，季节性波动
   - 长周期型（持续多年）：实用性强，可重复观看
4. **趋势对比**：2025-H1 vs 2025-H2 平均播放变化率

### 偏差风险

- 话题识别基于关键词匹配，可能误分类
- 峰值多由单个爆款贡献，非整体趋势

### 置信度判定

**中**：缺少增长时间序列验证

## 模式15：最佳发布时段分析

### 方法

```sql
-- 步骤1：按小时统计播放量
SELECT
    HOUR(FROM_UNIXTIME(timestamp)) as publish_hour,
    COUNT(*) as video_count,
    AVG(view_count) as avg_views,
    MAX(view_count) as max_views
FROM competitor_videos
WHERE timestamp IS NOT NULL
GROUP BY publish_hour
ORDER BY avg_views DESC;

-- 步骤2：按地区+小时交叉分析
SELECT
    CASE
        WHEN title LIKE '%台灣%' OR title LIKE '%台湾%' THEN '台湾'
        WHEN channel_title LIKE '%香港%' THEN '香港'
        ELSE '其他'
    END as region,
    HOUR(FROM_UNIXTIME(timestamp)) as publish_hour,
    AVG(view_count) as avg_views
FROM competitor_videos
GROUP BY region, publish_hour;
```
