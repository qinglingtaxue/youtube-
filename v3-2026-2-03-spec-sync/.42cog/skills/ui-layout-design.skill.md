# UI 布局设计技能

> **触发条件**：当用户要求设计页面布局、画布局草图、设计 UI 时使用
>
> **核心原则**：用户第一秒没看到想要的功能，就会划走不看了

---

## 设计流程

### 步骤 1：确认页面目标

回答以下问题：
- [ ] 这个页面的**核心用户任务**是什么？
- [ ] 用户来这个页面**最想做的一件事**是什么？
- [ ] 用户完成任务后**期望的结果**是什么？

### 步骤 2：确定首屏内容

按优先级排列首屏内容（从上到下）：

```
优先级 1：用户核心操作入口（搜索框、主按钮）
优先级 2：用户历史状态（搜索记录、最近使用）
优先级 3：核心功能入口（最多3个）
优先级 4：快速发现价值（推荐内容、热门）
优先级 5：统计概览（放最底部）
```

### 步骤 3：设计交互组件

对于每个交互组件，确认：

**搜索组件**：
- [ ] 搜索框是否是页面第一个可交互元素？
- [ ] 是否显示用户的搜索历史？
- [ ] 搜索历史是否紧跟搜索框下方？

**筛选组件**：
- [ ] 筛选条件是否默认折叠？
- [ ] 折叠状态是否显示已选条件数量？
- [ ] 筛选选项是否都是用户能理解的概念？

**排序组件**：
- [ ] 排序是否拆分为：时间范围 + 字段 + 方向？
- [ ] 是否用自然语言显示当前排序规则？
- [ ] 是否避免了"综合排序"等模糊概念？

**列表组件**：
- [ ] 每条结果是否显示完整的关键信息？
- [ ] 是否有分页或加载更多？

### 步骤 4：检查设计约束

**视觉层级检查**：
- [ ] 首屏第一个元素是用户的核心操作入口吗？
- [ ] 用户历史（搜索记录）放在首屏了吗？
- [ ] 统计数据/概览是否放在了页面底部？
- [ ] 功能入口是否控制在 3 个以内？

**用户认知检查**：
- [ ] 所有筛选/排序选项用户都能理解吗？
- [ ] 是否避免了专业术语（互动率、中心性、P分位数）？
- [ ] 复杂指标是否有括号解释？

**平台一致性检查**：
- [ ] 时长分类是否使用 YouTube 原生（<4min / 4-20min / >20min）？
- [ ] 时间范围是否使用 YouTube 原生（1h / 24h / 7d / 30d / 1yr）？

**用户控制检查**：
- [ ] 用户能自己选择排序字段吗？
- [ ] 用户能自己选择排序方向吗？
- [ ] 用户能自己选择时间范围吗？

**状态持久化检查**：
- [ ] 搜索历史是否会保存？
- [ ] 筛选条件是否会保存？
- [ ] 是否提供清空选项？

## 禁止做的事 ❌

### 视觉层级
- ❌ 把"数据概览"/"统计数字"放在首屏顶部
- ❌ 把次要功能（如采集设置）放在显眼位置
- ❌ 功能入口超过 3 个

### 筛选设计
- ❌ 默认展开所有筛选条件
- ❌ 在首屏显示超过 2 行的筛选选项
- ❌ 包含用户看不懂的筛选（如"互动率范围"）

### 排序设计
- ❌ 使用"综合排序"、"热门"、"推荐"等模糊概念
- ❌ 把时间范围、排序字段、排序方向混在一个下拉框
- ❌ 隐藏排序方向选择
- ❌ 不显示当前排序规则

### 分类分档
- ❌ 发明 YouTube 不支持的时长分档（如 0-5min, 5-10min）
- ❌ 发明 YouTube 不支持的时间范围（如 2天, 15天）
- ❌ 使用任意数值分档（如标题长度 15/30/50 字符）

## 用户反馈记录

以下是历史用户反馈，设计时应避免重复这些错误：

| 反馈 | 问题 | 正确做法 |
|------|------|---------|
| "用户第一秒没看到想要的功能就划走" | 统计数据放首屏顶部 | 搜索框放最上，统计放最下 |
| "记住用户搜索过什么" | 不保存搜索历史 | 保存并显示在搜索框下方 |
| "筛选条件可以折叠" | 默认展开所有筛选 | 默认折叠，显示已选数量 |
| "互动率用户看不懂" | 用专业术语做筛选 | 只用用户能理解的概念 |
| "排序不要写死" | 固定排序方式 | 让用户选字段+方向 |
| "从高到低还是从低到高" | 隐藏排序方向 | 独立的方向选择器 |
| "youtube不是这么筛选的" | 自定义时长分档 | 使用 YouTube 原生分类 |
| "综合排序是什么？" | 使用模糊概念 | 用自然语言说明规则 |

## 输出要求

设计完成后，必须附带：

1. **数据来源表**：每个区块的数据来自哪里
2. **约束标注**：已知的数据限制（如"无小时数据"）
3. **交互说明**：折叠/展开、点击跳转等交互
