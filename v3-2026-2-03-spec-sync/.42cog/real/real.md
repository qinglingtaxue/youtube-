# 现实约束 (Real) - 精简版

> 这些是在现实世界中会造成实际损失的硬性约束。
>
> **核心设计理念**：四条必选原则 + 三条备选原则 + 专题文档索引。
>
> 如需深入理解某个约束，请查阅对应的专题文档。

---

## 🔴 4 条必选原则（必须遵守）

### 原则 1：内容合规性（YouTube 社区准则 + 版权）

**红线规则**（触犯即下架/封号）：
- ❌ 不得批量上传重复/低质内容
- ❌ 不得使用误导性标题/封面
- ❌ 不得在元数据中堆砌无关关键词
- ❌ 竞品视频仅供分析，不得直接使用
- ❌ 背景音乐必须免版税或已授权
- ❌ 转载内容需注明来源

**执行时机**：发布前人工审核 + 制作阶段素材来源记录

**扩展阅读**：无（包含在本原则中）

---

### 原则 2：技术可行性约束（API 配额 + 自动化速率）

**硬性限制**：
- YouTube Data API **每日 10,000 单位**配额（需谨慎使用）
- 浏览器自动化**每次操作间隔 2-5 秒**（防被限流）
- 登录态 Cookie **定期刷新或重新登录**（有有效期）

**关键权衡**（三角约束）：
```
  速率 (采多少)
    /\
   /  \
  /    \  ← 三者不可兼得，根据阶段目标取舍
 /______\
质量    覆盖度
```

三种采集策略：
- **广度优先**：1000条/天，低质量，高覆盖 → 建立基线
- **深度优先**：100条/天，高质量，低覆盖 → 竞品深度分析
- **平衡模式**：500+50条/天，中质量，中覆盖 → 日常推荐

**执行时机**：调研/发布阶段配额计数

**扩展阅读**：`.42cog/real/real-data-collection.md`

---

### 原则 3：数据诚实性（避免无意义分析）

**核心约束**：任何分析分档都必须基于真实约束或业务含义，而非任意数字

**红线规则**（违反=虚假分析）：
- ❌ **禁止任意数值分档**（如标题 15/30/50 字符）
- ❌ **禁止模糊标签**（如"短"、"中"、"长"）
- ❌ **禁止不同内容混合分析**（教程 vs 段子时长完全不同）
- ❌ **禁止事后聚合伪装成先验约束**

**正确做法**：
- ✅ 使用 YouTube 原生分档（<4分、4-20分、>20分）
- ✅ 使用数据驱动 P-分位数（每组等量样本）
- ✅ 按内容类型分别分析（教程和教程对比）
- ✅ 保留原始数据和分布（不做聚合）

**执行时机**：任何数据分析之前

**扩展阅读**：`.42cog/real/real-analysis-quality.md`

---

### 原则 4：存储和成本管理

**硬性限制**：
- 视频文件体积大（1080p 约 100MB/分钟），**必须滚动清理**
- 竞品视频**30 天自动删除**（避免无限缓存）
- Token 消耗需监控，**综合模式单次 ~1,000 token**

**执行时机**：每日定时任务 + 制作阶段预算追踪

**扩展阅读**：无（包含在本原则中）

---

## 🟡 3 条备选原则（建议遵守，可根据上下文调整）

### 备选原则 1：发布时间优化

**建议做法**：
- 目标受众活跃时间发布（中文内容建议北京时间 18:00-22:00）
- 避免密集发布（同频道每日最多 1-2 个视频）

**执行时机**：发布阶段

---

### 备选原则 2：SEO 最佳实践

**建议做法**：
- 标题长度 50-60 字符（移动端显示完整）
- 描述前 150 字符应包含核心关键词（搜索结果可见）
- 标签 5-15 个（太多反而稀释相关性）

**执行时机**：策划阶段 seo_optimizer.py 评分

---

### 备选原则 3：本地环境依赖

**建议检查**：
- mcp-chrome 需要 Chrome 浏览器已安装并登录 YouTube
- Whisper 模型首次运行下载权重（base 模型 ~150MB）
- FFmpeg 需要支持 libx264 编码器

## 🔗 与其他文档的关系

**约束的执行流程**（从上游到下游）：

```
💡 创意阶段（无约束）
  ↓
📊 数据采集阶段 → 检查"原则 2"（API 配额）
                 → 参考 real-data-collection.md
  ↓
📈 数据分析阶段 → 检查"原则 3"（诚实性）
                 → 参考 real-analysis-quality.md
  ↓
🎨 UI 设计阶段  → 检查"原则 1"（合规性基础）
                 → 参考 real-ui-*.md（具体页面）
  ↓
📝 制作阶段     → 检查"原则 1"（版权、合规）
                 → 检查"原则 4"（存储管理）
  ↓
🚀 发布阶段     → 检查"原则 1"（最终审核）
                 → 参考"备选原则 1"（发布时间）
```

## ✅ 快速检查清单

在开始任何新的工作前，快速确认：

### 数据采集任务
- [ ] 是否已评估 API 配额消耗？
- [ ] 是否选择了正确的采集策略（广度/深度/平衡）？
- [ ] 是否了解了数据采集能力边界？
- → **参考**：real-data-collection.md

### 数据分析任务
- [ ] 分档依据是什么？（YouTube 原生 / P-分位数 / 业务含义）
- [ ] 是否混合了不同类型的内容？
- [ ] 时间粒度足够吗？（日期级 vs 小时级）
- → **参考**：real-analysis-quality.md

### UI 设计任务
- [ ] 是否遵循了工作记忆约束（Tab ≤ 5）？
- [ ] 首屏第一眼是核心操作入口吗？
- [ ] 是否使用了用户不理解的术语？
- [ ] 排序是否拆分为 3 个独立选择？
- → **参考**：real-ui-design.md（+ 具体页面文档）

### 发布任务
- [ ] 是否人工审核了合规性？
- [ ] 元数据是否符合 SEO 规范？
- [ ] 存储空间是否足够？
- → **参考**：原则 1 + 备选原则 2
