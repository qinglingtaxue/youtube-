 Bug 描述

 安装 42plugin 后，在 Claude Code 终端中输入 /plugin 命令无法识别，斜杠命令未注册成功。

 复现步骤

 1. 执行 42plugin install 42ailab/sci/psych-review --force 安装插件
 2. 执行 bun update -g @42ailab/42plugin 更新到最新版本 (v0.2.28)
 3. 执行 curl -fsSL https://get.42plugin.com/mac | bash 运行安装脚本
 4. 在 Claude Code 中输入 /plugin
 5. 尝试 42plugin builtin disable 42plugin-chat 和 42plugin builtin enable 42plugin-chat 后重启 Claude Code
 6. 尝试 42plugin builtin reset 后重启 Claude Code

 期望行为

 输入 /plugin 后应弹出插件管理菜单（类似终端中 curl 脚本显示的交互式菜单）。

 实际行为

 输入 /plugin 后，命令未被识别，直接显示为文本 /plugin，无任何响应。

 42plugin builtin list 显示 42plugin-chat 状态为「已启用」，但实际斜杠命令不可用。

 环境信息

 - 操作系统：macOS (Darwin 22.2.0)
 - Claude Code 版本：v2.0.55
 - Claude 模型：Opus 4.5 · Claude Max
 - 42plugin 版本：v0.2.28（已确认为最新版本）
 - 安装方式：bun 全局安装

 已尝试的解决方法

 1. 重新启用 42plugin-chat 插件并重启 Claude Code - 无效
 2. 执行 42plugin builtin reset 并重启 Claude Code - 无效
