什么时候适合用，什么时候不需要要用？

⏺ MCP 服务器使用场景分析

  浏览器自动化类

  @playwright/mcp

  适合用：
  - 需要与网页交互（点击、填表、截图）
  - 端到端测试、UI 自动化
  - 需要完整的页面快照和 DOM 访问
  - 处理 JavaScript 渲染的动态页面

  不需要用：
  - 只需要获取静态网页内容（用 WebFetch 即可）
  - 简单的 API 调用

  @hangwin/mcp-chrome

  适合用：
  - 需要控制用户已登录的 Chrome 浏览器
  - 访问需要登录态的页面
  - 利用现有的 cookies/session

  不需要用：
  - 公开页面访问（Playwright 或 WebFetch 更稳定）
  - 不需要登录态的场景

  ---
  信息检索类

  tavily

  适合用：
  - 需要实时网络搜索、新闻、最新信息
  - 研究特定话题、获取多来源信息
  - 需要搜索结果的摘要和排名

  不需要用：
  - 已有 WebSearch 内置工具可用
  - 查找特定已知 URL 的内容

  deepwiki

  适合用：
  - 查询开源项目/库的文档和用法
  - 理解 GitHub 仓库的架构和代码
  - 需要深入了解某个开源项目

  不需要用：
  - 本地代码分析（用 Grep/Glob/Read）
  - 非开源项目的问题

  mcp-server-fetch

  适合用：
  - 抓取网页原始内容
  - 需要更灵活的 HTTP 请求控制

  不需要用：
  - 已有内置 WebFetch 工具
  - 需要浏览器渲染的动态页面

  ---
  开发工具类

  github

  适合用：
  - 创建/管理 Issues、PR
  - 查看仓库信息、代码搜索
  - GitHub Actions 操作

  不需要用：
  - 本地 git 操作（用 Bash + gh 命令）
  - 简单的 git status/commit/push

  File System

  适合用：
  - 需要 MCP 标准化的文件操作接口

  不需要用：
  - Claude Code 已有内置的 Read/Write/Edit/Glob 工具，功能更强

  ---
  思考辅助类

  sequential-thinking

  适合用：
  - 复杂多步骤推理问题
  - 需要记录思考过程和中间状态
  - 数学证明、逻辑推导

  不需要用：
  - 简单直接的问题
  - 代码编写和调试（直接做更高效）

  Context7

  适合用：
  - 管理长对话中的上下文
  - 需要持久化特定信息

  不需要用：
  - 短对话或单次任务

  ---
  简单决策表

  | 场景            | 推荐工具                |
  |---------------|---------------------|
  | 读写本地文件        | 内置 Read/Write/Edit  |
  | 搜索代码          | 内置 Grep/Glob        |
  | 网页搜索          | 内置 WebSearch        |
  | 获取网页内容        | 内置 WebFetch         |
  | 网页交互/测试       | @playwright/mcp     |
  | 需登录态的网页       | @hangwin/mcp-chrome |
  | 开源项目文档        | deepwiki            |
  | GitHub API 操作 | gh 命令 或 github MCP  |
  | 复杂推理          | sequential-thinking |

  原则：优先使用内置工具，MCP 是补充而非替代。
