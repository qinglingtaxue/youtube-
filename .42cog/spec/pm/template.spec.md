# 创作模板规约 (Template Library Specification)

## 规约概述

本规约定义YouTube视频创作的模板库管理系统，确保创作模板能够高效生成、存储、复用和优化。

## 前置条件

### 模板来源
- 基于 `.42cog/cog/cog.md` 中定义的pattern实体
- 从成功案例中抽象提取
- 经过实战验证的创作框架

### 应用场景
- 直接套用创作
- 个性化定制
- 批量内容生产
- 培训教学使用

## 详细规约

### 1. 模板分类体系

#### 1.1 按创作模式分类

**M1：认知冲击型**
- **适用场景**：新号快速起量、颠覆常识观点
- **结构特点**：开头3秒制造震撼 → 核心观点论证 → 行动号召
- **模板示例**：
  ```
  【认知冲击开头】
  "你一直以为X是对的，但真相是..."
  
  【数据支撑】
  根据XX调查显示，90%的人都误解了...
  
  【观点输出】
  正确的做法应该是...
  
  【互动引导】
  你之前是怎么做的？评论区告诉我
  ```

**M2：故事叙述型**
- **适用场景**：建立情感连接、提升观看完播率
- **结构特点**：故事背景 → 冲突发展 → 解决升华
- **模板示例**：
  ```
  【故事背景】
  去年这个时候，我遇到了...
  
  【冲突设置】
  原本以为很简单，没想到...
  
  【解决过程】
  后来我尝试了...终于...
  
  【升华总结】
  这件事让我明白...
  ```

**M3：干货输出型**
- **适用场景**：打造专业形象、知识付费
- **结构特点**：问题提出 → 方法拆解 → 案例验证
- **模板示例**：
  ```
  【痛点切入】
  很多人都在问：如何...
  
  【方法论】
  我总结了3个步骤：
  1. 首先...
  2. 然后...
  3. 最后...
  
  【案例验证】
  拿XX举个例子...
  
  【总结行动】
  按照这个方法，你也能...
  ```

**M4：互动引导型**
- **适用场景**：提升算法权重、增加用户粘性
- **结构特点**：悬念设置 → 互动提问 → 引导关注
- **模板示例**：
  ```
  【悬念开头】
  看到最后有惊喜...
  
  【内容展示】
  （展示核心内容）
  
  【互动设置】
  1. 你觉得哪种方法最好？
  2. 你有什么更好的建议？
  
  【引导关注】
  关注我，每天分享...
  ```

#### 1.2 按内容领域分类

| 领域 | 模板数量 | 特点 | 示例模板 |
|------|----------|------|----------|
| 健康养生 | 15-20个 | 权威性强、注重科学性 | 医生科普、养生误区 |
| 教育培训 | 15-20个 | 逻辑清晰、实用性强 | 技能教学、知识分享 |
| 娱乐搞笑 | 15-20个 | 节奏紧凑、创意性强 | 段子脚本、模仿秀 |
| 商业财经 | 15-20个 | 专业度高、数据支撑 | 投资分析、商业模式 |

#### 1.3 按使用难度分类

**初级模板（★☆☆）**
- 结构简单，容易上手
- 适合新手创作者
- 成功率较高

**中级模板（★★☆）**
- 结构复杂，需要经验
- 适合有一定基础的创作者
- 效果较好

**高级模板（★★★）**
- 结构精妙，需要技巧
- 适合专业创作者
- 效果最佳

### 2. 模板结构规范

#### 2.1 标准模板结构

**T1：基础信息**
```yaml
template_id: "M1_001"
name: "认知冲击型开头"
category: "认知冲击"
difficulty: "★☆☆"
applicable_fields: ["健康", "教育", "商业"]
estimated_time: "15-20分钟"
success_rate: "75%"
```

**T2：模板内容**
```yaml
structure:
  - name: "开头模块"
    type: "hook"
    description: "制造认知冲击，吸引注意力"
    options:
      - "你一直以为X是对的，但真相是..."
      - "医生绝不会告诉你..."
      - "99%的人都误解了..."
    
  - name: "论证模块"
    type: "proof"
    description: "用数据和案例支撑观点"
    options:
      - "根据XX调查显示..."
      - "我分析了100个案例..."
      - "专家研究表明..."
    
  - name: "输出模块"
    type: "solution"
    description: "给出正确做法或观点"
    options:
      - "正确的做法是..."
      - "其实应该..."
      - "真正有效的是..."
    
  - name: "互动模块"
    type: "engagement"
    description: "引导用户参与互动"
    options:
      - "你之前是怎么做的？"
      - "评论区说说你的经历"
      - "觉得有用的话点个赞"
```

**T3：使用说明**
```yaml
instructions:
  - step: "1"
    action: "选择合适的开头方式"
    tips: "根据目标受众选择最震撼的开头"
  
  - step: "2"
    action: "填充具体内容"
    tips: "确保数据真实，案例典型"
  
  - step: "3"
    action: "优化语言表达"
    tips: "语言要通俗易懂，避免专业术语"
  
  - step: "4"
    action: "添加互动元素"
    tips: "在关键位置设置互动点"
```

#### 2.2 高级模板结构

### 3. 模板生成流程

#### 3.1 自动生成

**输入条件**
- 主题关键词
- 目标受众
- 内容长度
- 创作目标

**生成步骤**
1. **模式匹配**
   - 根据关键词匹配合适的创作模式
   - 评估模式的适用度

2. **结构生成**
   - 自动选择模板结构
   - 填充基础框架

3. **内容优化**
   - 智能推荐开头方式
   - 提供论证素材建议

4. **个性调整**
   - 根据受众特点调整语言
   - 匹配内容领域特征

#### 3.2 手动定制

**定制选项**
- 修改模板结构
- 替换模块内容
- 调整逻辑顺序
- 添加个性化元素

**质量控制**
- 实时语法检查
- 逻辑完整性验证
- 效果预测分析
