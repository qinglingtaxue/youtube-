# 决策10：确定信息采集分组与指标图示化方案

> 产品层决策 | 2026-01-19 | 来源：a437325d

## 决策背景

### 问题识别
- 采集的数据需要有组织地呈现
- 原始数据难以直接产生洞察
- 用户需要可视化的数据分析

### 触发因素
- 数据采集能力已建立
- 需要将数据转化为洞察
- 用户期望直观的数据展示

## 决策内容

### 核心决策
**建立信息采集分组机制，实现指标分析结果的图示化呈现**

### 信息采集分组

| 分组 | 包含指标 | 更新频率 |
|------|----------|----------|
| **基础信息** | 标题、封面、时长、发布时间 | 一次性 |
| **播放数据** | 播放量、观看时长、完播率 | 每日 |
| **互动数据** | 点赞、评论、分享、收藏 | 每日 |
| **频道数据** | 订阅量、视频数、总播放量 | 每周 |
| **增长数据** | 日增播放、周增订阅 | 实时计算 |

### 指标图示化方案

**1. 数据卡片**
```
┌─────────────────┐
│ 总视频数        │
│    1,020       │
│ ↑ 15% vs 昨日  │
└─────────────────┘
```

**2. 趋势图表**
- 播放量趋势（折线图）
- 分类分布（饼图）
- 时长分布（柱状图）

**3. 排行榜**
- 播放量 Top 50
- 增速 Top 20
- 互动率 Top 20

**4. 洞察结论**
```
┌─────────────────────────────────────┐
│ 📊 本周洞察                         │
├─────────────────────────────────────┤
│ • 短视频(<1min)占比提升 12%         │
│ • 游戏类内容平均播放量最高          │
│ • 周末发布的视频互动率更高          │
└─────────────────────────────────────┘
```

### 仪表盘布局
```
┌───────────────────────────────────────┐
│ 数据概览卡片（4个）                    │
├───────────────────────────────────────┤
│ 播放趋势图        │ 分类分布图         │
├───────────────────┼───────────────────┤
│ 爆款 Top 10       │ 潜力 Top 10        │
├───────────────────────────────────────┤
│ 本周洞察结论                          │
└───────────────────────────────────────┘
```

## 决策理由

### 支持因素
- 数据分组便于理解和维护
- 可视化提升数据价值
- 洞察结论降低分析门槛

### 设计原则
- 一眼看到关键指标
- 深入探索有路径
- 结论自动生成
