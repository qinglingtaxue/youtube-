# 决策09：设计视频/频道榜单筛选维度

> 产品层决策 | 2026-01-19 | 来源：7c1278df

## 决策背景

### 问题识别
- 视频和频道数据维度多样
- 用户有不同的筛选需求
- 需要提供灵活的数据探索能力

### 触发因素
- 视频标签体系已建立
- 需要更细粒度的筛选能力
- 用户需要根据业务场景定制视图

## 决策内容

### 核心决策
**设计多维度筛选体系，支持播放量、地区、时长等维度**

### 视频筛选维度

| 维度 | 选项 | 说明 |
|------|------|------|
| **播放量** | 1万+/10万+/100万+/1000万+ | 分级筛选 |
| **时长** | 短视频/中视频/长视频 | <1min/1-10min/>10min |
| **发布时间** | 24h/7d/1m/3m | 新鲜度筛选 |
| **地区** | 全球/美国/日本/... | 来源地区 |
| **语言** | 英语/中文/日语/... | 内容语言 |
| **分类** | 游戏/教育/娱乐/... | YouTube 分类 |

### 频道筛选维度

| 维度 | 选项 | 说明 |
|------|------|------|
| **订阅量** | 1万+/10万+/100万+ | 频道规模 |
| **视频数** | 10+/100+/1000+ | 内容丰富度 |
| **更新频率** | 日更/周更/月更 | 活跃度 |
| **主要地区** | 全球/美国/日本/... | 受众地区 |

### 筛选器 UI 设计
```
┌─────────────────────────────────────────────┐
│ 筛选条件                                     │
├─────────────────────────────────────────────┤
│ 播放量: [全部 ▼]  时长: [全部 ▼]             │
│ 时间:   [7天 ▼]   地区: [全部 ▼]             │
│ 分类:   [全部 ▼]  排序: [播放量 ▼]           │
└─────────────────────────────────────────────┘
```

### 组合筛选
- 支持多维度组合筛选
- 筛选条件可保存为预设
- 支持 URL 参数持久化

## 决策理由

### 支持因素
- 覆盖主要业务场景
- 维度相互独立可组合
- 便于用户快速定位目标

### 设计原则
- 常用维度优先展示
- 默认值符合多数场景
- 支持快速重置
